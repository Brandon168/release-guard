# Repository Grounding Pack

This folder is the committed, repo-specific context pack for Release Guard.

The goal is to keep project context separate from the model choice. Any model can
be swapped in as long as it receives the same grounding pack, deterministic
baseline, and change artifact.

## What Belongs Here

- `project.md`: what this repo does, how it is deployed, and what is in scope
- `risk-taxonomy.md`: repo-local definitions for low, medium, high, and unknown
- `policy.yaml`: path sensitivity, escalation rules, and policy defaults
- `examples/`: short repo-native examples that calibrate the grader

`policy.yaml` is also where repo owners can define PR exemptions for non-runtime
files. For example, docs-only or `.changeRisk/`-only changes can bypass
operational rollout and rollback requirements if the repo policy says so.

## Demo Story

In a production setup, this pack could be maintained by humans, generated by a
repo bootstrapping skill, or refreshed by a background job that scans the codebase
and proposes updates for review.

For this take-home demo, the pack is intentionally small and readable so a reviewer
can see exactly what the model is being grounded on.
